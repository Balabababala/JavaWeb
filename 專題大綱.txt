1. users – 使用者表（合併前台和後台用戶）
id BIGINT (PK)                使用者 ID
username VARCHAR(255) UNIQUE  使用者名稱
hash_password VARCHAR(255)    密碼（加密）
hash_salt		      加鹽
email VARCHAR(255) UNIQUE     電子郵件
role_id BIGINT (FK)           角色 ID（連接到 roles 表）
created_at DATETIME           建立時間
last_login_at DATETIME        最後登入時間（可選）

2. products – 商品表
id BIGINT (PK)                商品 ID
name VARCHAR(255)             商品名稱
description TEXT              商品描述
price DECIMAL(10,2)           價格
stock INT                     庫存數量
image_url TEXT                商品圖片連結
category_id BIGINT (FK)       所屬分類
created_at DATETIME           上架時間（可選）

3. categories – 商品分類表
id BIGINT (PK)                分類 ID
name VARCHAR(100) UNIQUE      分類名稱
parent_id BIGINT (FK)         父分類 ID（可選，指向自身）

4. cart_items – 購物車明細
id BIGINT (PK)                購物車項目 ID
user_id BIGINT (FK)           所屬使用者
product_id BIGINT (FK)        商品 ID
quantity INT CHECK (>0)       數量
added_at DATETIME             加入時間（可選）

5. orders – 訂單主表
id BIGINT (PK)                訂單 ID
user_id BIGINT (FK)           購買人
seller_id BIGINT	      賣方
order_date DATETIME           下單時間
status ENUM(
  'PENDING',           -- 等待付款
  'PAID',              -- 已付款，待出貨
  'SHIPPED',           -- 已出貨
  'DELIVERED',         -- 已收貨
  'COMPLETED',         -- 訂單完成
  'CANCELLED',         -- 訂單取消
  'RETURN_REQUESTED',  -- 退貨申請中
  'RETURNED'           -- 退貨完成
)) 訂單狀態
shipping_method	VARCHAR(100)	配送方式（例如：宅配、超商取貨）
payment_method	VARCHAR(100)	付款方式（例如：信用卡、貨到付款）
payment_status	ENUM('PENDING', 'PAID', 'FAILED', 'REFUNDED')	付款狀態 (有些時候「付款狀態」和「訂單狀態」不一致，比如：
								付款失敗（payment_status = FAILED），但訂單還留在 PENDING。
								已付款（payment_status = PAID），但訂單還沒出貨（status = PAID）。
								成功退款了（payment_status = REFUNDED），訂單仍然要顯示 RETURNED。)
tracking_number	VARCHAR(255)	物流單號（出貨後填入）
receiver_name	VARCHAR(255)	收件人姓名
receiver_phone	VARCHAR(20)	收件人電話
total_amount DECIMAL(10,2)    訂單總金額
shipping_address TEXT         收貨地址（可選）

6. order_items – 訂單明細
id BIGINT (PK)                明細 ID
order_id BIGINT (FK)          所屬訂單
product_id BIGINT (FK)        商品 ID
quantity INT CHECK (>0)       數量
unit_price DECIMAL(10,2)      商品單價

1. roles – 角色表
id BIGINT (PK)                角色 ID
name VARCHAR(100) UNIQUE      角色名稱（User, Admin, Customer Support）
description TEXT              角色描述
created_at DATETIME           建立時間（可選）

2. permissions – 權限表
id BIGINT (PK)                權限 ID
name VARCHAR(100) UNIQUE      權限名稱（view_orders, manage_products, manage_users）
description TEXT              權限描述

3. role_permissions – 角色權限對應表
role_id BIGINT (FK)           角色 ID
permission_id BIGINT (FK)     權限 ID
assigned_at DATETIME          分配時間（可選）
